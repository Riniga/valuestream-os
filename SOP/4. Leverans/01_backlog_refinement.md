# SOP 1: Backlog refinement

## 1. Syfte
Förtydliga och bryta ner kommande arbete så att teamet kan planera och leverera effektivt och enligt definition of ready.



## 2. Kontext
- Processteg: 4. Leverans
- Delprocess: 1: Backlog refinement


## 4. Input
- Prioriterad backlog (epics, features, stories)
- Feedback från föregående iteration
- Tekniska insikter och beroenden


## 5. Output
- Förtydligade user stories
- Ready‑markerad backlog


## 6. RACI
- R: Produktägare
- A: Produktägare
- C: Business Analyst, Utvecklingsteam
- I: 


## 7. Arbetssteg
1. Förtydliga user stories genom dialog med teamet.
2. Bryta ner stories i mindre uppgifter där det behövs.
3. Definiera acceptance criteria för alla stories.
4. Säkerställa att stories följer INVEST‑principen.
5. Estimera stories tillsammans med teamet.
