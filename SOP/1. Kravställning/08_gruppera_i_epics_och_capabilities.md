# SOP 8: Gruppera i epics & capabilities

## 1. Syfte
Skapa en hanterbar struktur över funktionalitet genom att paketera user stories i större områden.

## 2. Primär roll
- Produktägare

## 3. Stödroller
- Business Analyst

## 4. Input
- Story Map
- User Stories

## 5. Arbetssteg
1. Identifiera övergripande capabilities.
2. Gruppera user stories under varje epic.
3. Säkerställa logisk koppling mellan user journeys och epics.
4. Validera paketindelningen med teamet.

## 6. Output
- Capabilities
- Epics

## 7. Output lämnas till
- Processteg: Prioritering (hög nivå)
