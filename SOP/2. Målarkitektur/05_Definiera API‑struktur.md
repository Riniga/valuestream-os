# SOP 5: Definiera API‑struktur

## 1. Syfte
Skapa en strukturerad API‑modell som styr hur system kommunicerar genom kontrakt och standarder.

## 2. Primär roll
- Lösningsarkitekt

## 3. Stödroller
- Utvecklingsrepresentanter
- Integrationsspecialist

## 4. Input
- Integrationsarkitektur
- Domänmodell

## 5. Arbetssteg
1. Definiera API‑typ (REST, GraphQL etc.).
2. Definiera API‑kontrakt baserat på domänobjekt.
3. Dokumentera standarder (versionering, formatering, felhantering).
4. Stämma av med utvecklingsteam före fastställande.

## 6. Output
- API‑struktur
- API‑kontrakt (hög nivå)

## 7. Output lämnas till
- Processteg: Datamodell
- Processteg: Arkitekturprinciper
