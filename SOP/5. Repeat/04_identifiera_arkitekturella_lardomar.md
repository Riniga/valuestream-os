# SOP 4: Identifiera arkitekturella lärdomar

## 1. Syfte
Identifiera tekniska och arkitekturella insikter som ska användas i nästa Målarkitekturs‑iteration.



## 2. Kontext
- Processteg: 5. Repeat
- Delprocess: 4: Identifiera arkitekturella lärdomar


## 4. Input
- Tekniska problem som uppstått
- Avvikelser från arkitekturprinciper
- Drift och integrationsproblem
- Output från Utvärdera leveransresultat


## 5. Output
- Arkitekturinsikter
- Lista över tekniska förbättringsområden


## 6. RACI
- R: Lösningsarkitekt
- A: Lösningsarkitekt
- C: Teknisk Lead
- I: 


## 7. Arbetssteg
1. Identifiera tekniska hinder som påverkat leveransen.
2. Notera avvikelser från arkitekturprinciper.
3. Dokumentera arkitekturella konsekvenser.
4. Skapa input till nästa målarkitektur (utan att ändra målbilden här).
