# Roller nödvändiga för att ta fram Roadmap och Leveransstrategi

```mermaid
flowchart LR
    %% SEKVENTIELL HUVUDFLÖDE
    A1["Skapa MVP"] --> A2["Strukturera story map till leveranspaket"]
    A2 --> A3["Identifiera tekniska beroenden"]
    A3 --> A4["Prioritera funktionalitet"]
    A4 --> A5["Skapa releaseplan"]
    A5 --> A6["Skapa produktroadmap"]

    %% VISUELL PARALLELLITET
    %% Prioritering kan påbörjas innan beroenden är KLARA, men inte avslutas.
    A3 -. delvis parallellt .-> A4

    %% STYLING
    classDef seq fill:#1e88e5,stroke:#0d47a1,color:#fff;
    classDef par fill:#43a047,stroke:#1b5e20,color:#fff;

    class A1,A2,A3,A4,A5,A6 seq;
```

## RA-Diagram: Vem utför och vem godkänner

```mermaid
flowchart LR
    subgraph " "
        A1["Skapa MVP"]
        A2["Strukturera story map till leveranspaket"]
        A3["Identifiera tekniska beroenden"]
        A4["Prioritera funktionalitet"]
        A5["Skapa releaseplan"]
        A6["Skapa produktroadmap"]
    end

    %% ROLLER (ENDAST DE SOM HAR R)
    BA["Business Analyst"]
    TL["Teknisk Lead"]
    LA["Lösningsarkitekt"]
    PA["Beställare"]


    BA -- R --> A1
    BA -- R --> A2
    BA -- R --> A3
    BA -- R --> A4
    BA -- R --> A5
    TL -- R --> A6

    A1 -- A --> PA
    A2 -- A --> PA
    A3 -- A --> PA
    A4 -- A --> PA
    A5 -- A --> PA
    A6 -- A --> LA

    %% STYLING
    classDef seq fill:#1e88e5,stroke:#0d47a1,color:#fff;
    classDef par fill:#43a047,stroke:#1b5e20,color:#fff;
    classDef slut fill:#6a1b9a,stroke:#4a148c,color:#fff;

    class A1,A2,A3,A4 seq;
    class B1,B2,B3 par;
    class C1,C2,C3 slut;
```
