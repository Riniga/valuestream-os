# SOP 3: Utveckling och kodgranskning

## 1. Syfte

Implementera funktionalitet enligt definierade krav, arkitektur och kvalitetsprinciper och kvalitetssäkra resultatet i samma utvecklarsittning innan det lämnas vidare till test.

## 2. Kontext

- Processteg: Leverans
- Delprocess: Utveckling och kodgranskning

## 3. Input

- Sprint backlog
- Arkitekturprinciper
- Målarkitektur

## 4. Output

- Produktinkrement

## 5. RACI

- R: Utvecklare
- A: Teknisk Lead
- C: Lösningsarkitekt
- I: Beställare

## 6. Arbetssteg

1. Tolka user stories och tillhörande acceptance criteria.
2. Implementera funktionalitet enligt arkitekturprinciper och riktlinjer.
3. Implementera integrationer och nödvändig backend-/frontend-logik.
4. Säkerställ att kod följer standarder och konventioner.
5. Utför egen kodgranskning eller pargranskning mot kodstandarder och arkitekturprinciper.
6. Åtgärda identifierade förbättringar direkt i produktinkrementet.
7. Utför lokal testning av funktionaliteten.
8. Dokumentera tekniska förändringar vid behov.
9. Markera produktinkrementet som klart för test.
