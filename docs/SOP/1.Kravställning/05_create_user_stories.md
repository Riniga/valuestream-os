# SOP 5: Create User Stories

## 1. Syfte

Skapa en första strukturerad uppsättning user stories baserat på mål, behov och aktörer, för att möjliggöra vidare analys (user journeys, domänmodell och arkitektur).

---

## 2. Kontext

- Processteg: Kravställning
- Delprocess: Create User Stories

---

## 3. Input

- Vision & målbild
- Affärsmål & värdebild
- Scope & avgränsningar
- Stakeholderkarta

---

## 4. Output

- User Stories

---

## 5. RACI

- R: Business Analyst
- A: Produktägare
- C: Verksamhetsexperter, Användarrepresentanter
- I: Lösningsarkitekt, Utvecklare

---

## 6. Arbetssteg

1. Identifiera aktörer (användartyper) baserat på stakeholderkartan.
2. Identifiera centrala behov/problem kopplade till varje aktör.
3. Formulera user stories i enkel form:
   “Som [aktör] vill jag [behov] så att [värde]”
4. Säkerställ att varje story kopplar till ett affärsmål eller värde.
5. Undvik tekniska lösningar — fokusera på behov och intention.
6. Identifiera dubbletter och slå ihop liknande stories.
7. Säkerställ rimlig granularitet (inte för stora, inte för detaljerade).
8. Validera stories med verksamhet och produktägare.
9. Dokumentera stories i gemensamt artefakt (User Stories).

---

## Notering

User Stories i detta steg är initiala och kommer att förfinas i Leverans (refinement).
