# SOP 5: Definiera API-struktur

## 1. Syfte

Definiera en konsekvent API-struktur och kontrakt som möjliggör tydlig och standardiserad kommunikation mellan system.

## 2. Kontext

- Processteg: Målarkitektur
- Delprocess: Definiera API-struktur

## 3. Input

- Integrationsarkitektur
- Domänmodell

## 4. Output

- API-specifikation

## 5. RACI

- R: Lösningsarkitekt
- A: Enterprise Arkitekt
- C: Teknisk Lead, Utvecklare
- I: Produktägare

## 6. Arbetssteg

1. Identifiera API-behov baserat på integrationsarkitektur och domänmodell.
2. Definiera API-typ (t.ex. REST, GraphQL, event-driven).
3. Strukturera API-endpoints och resurser baserat på domänobjekt.
4. Definiera API-kontrakt på hög nivå (request/response, datamodeller).
5. Fastställ standarder för versionering, format och felhantering.
6. Säkerställ att API-strukturen stödjer integrationsarkitektur och arkitekturmål.
7. Validera API-struktur med teknisk lead och utvecklare.
8. Fastställ API-struktur och kontrakt (beslut av A).
